# Bland

**Bland** is an opinionated, minimal Ubuntu setup.

Reinstalling a system is usually an emotional process: too many choices, too much noise, and too many tools trying to be clever.  
**Bland** exists to remove that friction by making system setup boring, predictable, and explicit.

## What Bland is

- A reproducible way to install **my default Ubuntu environment**
- A collection of simple shell scripts
- A documented set of system and software choices

Everything in **Bland** is intentional. There are no abstractions, no frameworks, and no hidden logic.

## What Bland is not

- A generic installer for everyone
- A configuration framework
- A TUI-based setup wizard
- A replacement for tools like Ansible, Nix, or cloud provisioning systems

## Design principles

- **Boring is good** — predictable beats clever  
- **Explicit over abstract** — scripts should be readable without context  
- **Editability first** — modifying the scripts is the intended workflow  
- **No magic** — what runs is exactly what you see  

## Customization

**Bland** ships with my personal defaults.  
Customizing it means editing the scripts directly.

Any power user with basic shell experience can read, modify, and adapt **Bland** to their own needs. Forking is expected.

## Intended audience

- People who prefer simple shell scripts over frameworks
- Developers who want a clean, reproducible personal setup
- Anyone looking for a practical reference for system bootstrap scripting

## Why Bland

Because system setup doesn’t need personality, opinions, or interaction.  
It just needs to work the same way every time.

No hype. No magic. Just a boring system setup.
